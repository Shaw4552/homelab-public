# VPN Architecture

The lab uses multiple VPN models for different purposes.

## Remote Access VPN
Used for secure administrative and user access to internal resources.

## Site-to-Site VPN
Used to connect two locations and extend access to selected internal services.

## Policy-Based Routing
Used to direct selected traffic through specific VPN egress paths based on network role or use case.

## Design Principles

- VPN complements firewall segmentation
- remote access is limited to required resources
- inter-site access is intentionally scoped
